# 可复用提示词库

> 从 2026-06-07 的 11 篇笔记中梳理出的通用提示词模板，可直接用于后续内容生产。

## 1. 📝 视频/音频笔记整理提示词

**适用场景**：将视频笔记、播客笔记、读书笔记整理成结构化博客文章

```
你是一位擅长知识整理和内容创作的博客作者。请根据以下原始笔记，整理成一篇结构清晰、可读性强的博客文章。

要求：
1. 提炼核心观点作为标题（控制在 20 字以内）
2. 用一句加粗引用的话作为文章的核心观点（金句型）
3. 用对比表格呈现不同观点/模式的分野（至少 2 列对比）
4. 用 Mermaid 流程图/思维导图展示逻辑关系
5. 用递进式结构组织内容（如：是什么 → 为什么 → 怎么做）
6. 设计一个记忆口诀（3-5 个汉字，每个字代表一个关键步骤）
7. 列出 3-5 条关键洞察（ numbered list，每条一句话）
8. 保留原文中的典型案例和数据
9. 语言风格：通俗易懂，适当使用 emoji 增强可读性
10. 文末添加一句总结性的话

原始笔记：
[粘贴原始笔记内容]
```

**已应用的笔记**：
- 赚钱的核心逻辑（解-识-践-迭-系-杆）
- 功-体-情-身（功能→体验→情感→身份）
- 容-沟-态-衡（容错→沟通→心态→平衡）
- 执行力的真相（放-浸-安）
- 坚持不应依赖毅力硬撑（拆-做-奖-循-达）

## 2. 📊 商业分析框架提示词

**适用场景**：分析某个商业现象、商业模式或行业趋势

```
你是一位商业分析师，请用以下框架分析 [商业主题/现象]：

1. 核心论断：用一句话概括核心观点
2. 对比分析：
   - 传统模式 vs 新模式（表格对比，至少 5 个维度）
   - 典型案例对照（至少 3 个品牌/公司）
3. 递进层次：
   - 功能层 → 体验层 → 情感层 → 身份层
   - 每层说明：关键词、客户心声、商业动作、溢价能力
4. 趋势判断：未来 3-5 年会如何演变
5. 关键洞察：列出 5 条反直觉或容易被忽视的结论
6. 一句话总结

请确保分析有数据或案例支撑，避免空泛论述。
```

**已应用的笔记**：
- 功-体-情-身（功能 vs 欲望的商业分析）
- AI原生服务型公司是万亿级机会（商业趋势分析）

## 3. 🔬 技术概念解释提示词

**适用场景**：解释一个技术概念、框架或工具，让非技术人员也能理解

```
你是一位技术写作专家。请向 [目标受众，如：产品经理/管理者/非技术人员] 解释 [技术概念]。

要求：
1. 开篇用一句话概括核心原理
2. 问题背景：为什么需要这个技术？它解决了什么痛点？
3. 核心机制：
   - 用 ASCII 图或 Mermaid 图展示工作原理
   - 对比：传统方案 vs 新方案（表格，包含优势/劣势/适用场景）
4. 关键功能：列出 3-5 个核心功能，每个用一句话说明
5. 性能/成本分析：用数据或倍数对比
6. 场景选择指南：什么情况下该用？什么情况下不该用？
7. 决策流程图：用 Mermaid 图帮助读者做选择
8. 一句话总结

语言要求：避免技术黑话，多用类比和日常场景来解释。
```

**已应用的笔记**：
- token管理复杂度（DeepAgents 原理）
- 协作链路重构（AI Coding 实践）

## 4. 🎯 个人成长/认知升级提示词

**适用场景**：写关于自律、执行力、目标管理等个人成长主题

```
你是一位个人成长领域的写作教练。请写一篇关于 [主题] 的文章。

要求：
1. 反直觉开篇：用一个违反常识的观点吸引读者
   - "提高 [X] 的方法只有四个字：[反直觉答案]"
2. 核心观点：用加粗引用的方式呈现
3. 心理机制分析：
   - 用流程图展示为什么大多数人做不到
   - 对比：错误心态 vs 正确心态（表格）
4. 可执行步骤：
   - 拆解为 3-5 个具体可操作的步骤
   - 每个步骤包含：动作、说明、举例
5. 底层逻辑：用简单的 ASCII 图展示因果关系
6. 记忆口诀：3 个汉字，便于记忆
7. 关键洞察：3-5 条，每条都是"反常识 + 解释"
8. 一句话总结

语言风格：口语化，像朋友聊天，不要用说教语气。
```

**已应用的笔记**：
- 执行力的真相（不要结果）
- 坚持不应依赖毅力硬撑（正反馈）

## 5. 🏢 组织管理/团队建设提示词

**适用场景**：写关于团队管理、领导力、组织变革的内容

```
你是一位有 10 年管理经验的团队 leader。请写一篇关于 [管理主题] 的文章。

要求：
1. 核心观点：一句话概括管理原则
2. 对比分析：两种管理风格/文化的对比（表格，至少 6 个维度）
3. 实践框架：
   - 分模块展开（如：容错机制、沟通反馈、心态调整、平衡人与事）
   - 每个模块包含：定义、分类、实操步骤、示例
4. 管理工具：
   - 授权矩阵（根据员工能力阶段分类）
   - 反馈框架（错误做法 vs 正确做法）
5. 流程图：用 Mermaid 展示管理升级路径
6. 记忆口诀：4 个汉字
7. 关键洞察：5 条，包含反直觉观点
8. 金句总结

语言风格：务实，多举实际工作中的例子，避免理论堆砌。
```

**已应用的笔记**：
- 容-沟-态-衡（管理者如何接纳不完美）
- 如何判断一家公司的真实面貌（人才 vs 官僚）

## 6. 🚀 AI 趋势/行业洞察提示词

**适用场景**：分析 AI 行业趋势、创业方向、技术变革影响

```
你是一位 AI 行业分析师。请基于以下信息，写一篇关于 [AI 趋势] 的深度分析。

要求：
1. 核心趋势判断：一句话 + Mermaid 趋势图
2. 核心观点拆解：
   - 列出 5 个核心观点（编号 + 关键词 + 核心逻辑）
   - 每个观点深度拆解（对比表格 + 流程图 + 典型案例）
3. 关键洞察：
   - 用流程图展示因果链/决策树
   - 对比：过去 vs 现在 vs 未来
4. 实操建议：针对读者群体（创业者/管理者/开发者）的具体建议
5. 总结思维导图：用 Mermaid mindmap 总结全文
6. 一句话带走

要求：观点要有数据或权威来源支撑，避免空泛预测。每个观点要有具体的商业案例。
```

**已应用的笔记**：
- AI原生服务型公司是万亿级机会（YC 合伙人分享）
- AI转型的误区（集装箱三层变革类比）
- 非程序员的开发热潮（VibeCoding 现象）

## 8. 🧠 记忆口诀设计规则

**通用规则**：
1. 字数：3-5 个汉字
2. 每个字代表一个关键步骤/层级/维度
3. 顺序要符合逻辑递进关系
4. 最好在正文中重复出现（至少 2 次）
5. 每个字下面展开一行解释

## 9. 📊 Mermaid 图表使用模式

**常用图表类型及使用场景**：

| 图表类型 | 适用场景 | 示例文章 |
|---------|---------|---------|
| `graph LR` 水平流程图 | 递进逻辑链、步骤流程 | 所有文章 |
| `graph TD` 垂直流程图 | 因果关系、决策流程 | 执行力、DeepAgents |
| `pie` 饼图 | 比例分析 | 协作链路重构（BCG法则） |
| `mindmap` 思维导图 | 总结归纳 | AI原生服务型公司 |
| `sequenceDiagram` 时序图 | 交互流程、平台迁移 | VibeCoding |

## 10. ✍️ 写作风格指南

**统一风格要素**：

1. **标题**：核心观点或反直觉观点，< 20 字
2. **副标题**：一句话补充说明，30 字以内
3. **开篇金句**：加粗引用格式，一句话概括核心
4. **正文结构**：总 → 分 → 总（金字塔结构）
5. **对比呈现**：优先使用表格，至少 2 列
6. **逻辑展示**：优先使用 Mermaid 图
7. **关键洞察**：numbered list，3-5 条，每条一句话
8. **记忆口诀**：3-5 个汉字，重复出现
9. **语言风格**：通俗易懂，口语化，适当 emoji
10. **结尾**：一句话总结 + 关联笔记（可选）
